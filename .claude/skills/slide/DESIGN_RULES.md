# デザインルール - /slide Skill

このドキュメントは、スライド生成時に適用すべきデザイン原則を定義しています。

---

## 1. 余白とグリッド

### 余白（Whitespace）
- **最小マージン**: 60px（上下）、80px（左右）
- **要素間の余白**: 最低20px
- **セクション間**: 40px以上

### なぜ余白が重要か
- 視認性の向上
- プロフェッショナルな印象
- 情報の整理と区切り

### 実践ルール
```css
section {
  padding: 60px 80px;
}
```

❌ NG: 画面いっぱいにテキストを詰め込む
✅ OK: 余白を十分に取り、呼吸できるレイアウト

## 2. 色彩理論（カラーパレット）

### メインカラーパレット

| 用途 | カラーコード | 使用場面 |
|------|-------------|----------|
| Primary | `#2563EB` | 見出し、アクセント、CTA |
| Secondary | `#64748B` | サブテキスト、補足情報 |
| Accent | `#F59E0B` | 強調、警告、注目点 |
| Background | `#FFFFFF` | メイン背景 |
| Background Alt | `#F8FAFC` | セクション区切り |
| Text | `#1E293B` | 本文テキスト |
| Text Light | `#64748B` | キャプション、注釈 |

### カラー使用ルール

1. **60-30-10ルール**
   - 60%: 背景色（白/薄いグレー）
   - 30%: メインテキスト色
   - 10%: アクセントカラー

2. **コントラスト比**
   - テキストと背景のコントラスト比は4.5:1以上
   - 重要な情報は7:1以上

3. **色の意味**
   - 青系（#2563EB）: 信頼、プロフェッショナル
   - オレンジ系（#F59E0B）: 注目、行動喚起
   - グレー系: 補足、サブ情報

### タイトルスライド用ダークテーマ
```yaml
<!-- _backgroundColor: #2563EB -->
<!-- _color: white -->
```

## 3. タイポグラフィ

### フォントスタック
```css
font-family: 'Noto Sans JP', 'Hiragino Sans', 'Yu Gothic', sans-serif;
```

### フォントサイズ階層

| 要素 | サイズ | 用途 |
|------|--------|------|
| h1（タイトル） | 2.5em (40px) | スライドタイトル |
| h2（見出し） | 1.8em (28px) | セクション見出し |
| p/li（本文） | 1.3em (20px) | 箇条書き、説明 |
| small（注釈） | 0.9em (14px) | キャプション、出典 |

### フォントウェイト
- **見出し**: Bold (700)
- **本文**: Regular (400)
- **強調**: Bold (700) + Primary Color

### 行間（Line Height）
- 見出し: 1.3
- 本文: 1.8（読みやすさ重視）
- 箇条書き: 1.6

### タイポグラフィのNG例
❌ 複数のフォントファミリーを混在させる
❌ フォントサイズが小さすぎる（16px未満）
❌ 長い文章を1行に詰め込む
❌ 全て大文字/全て太字

## 4. 視覚的階層

### 情報の優先順位付け

```
[最重要] タイトル・キーメッセージ
    ↓
[重要] 見出し・セクションタイトル
    ↓
[通常] 本文・箇条書き
    ↓
[補足] 注釈・出典・ページ番号
```

### 視覚的階層の作り方

1. **サイズ**: 重要なものは大きく
2. **色**: 重要なものにアクセントカラー
3. **位置**: 重要なものは上部・中央に
4. **余白**: 重要なものの周りに余白を

### フォーカルポイント
- 各スライドに**1つの主役**を設定する
- 主役以外は控えめなスタイル
- 視線の流れ: 左上 → 右下（Zパターン）

## 5. レイアウトパターン

### パターン1: タイトルスライド
```markdown
<!-- _class: lead -->
<!-- _backgroundColor: #2563EB -->
<!-- _color: white -->

# 🚀 プレゼンテーションタイトル

**サブタイトルやキャッチコピー**

登壇者名 | 日付
```

### パターン2: コンテンツスライド
```markdown
# セクションタイトル

- 📌 ポイント1: 説明テキスト
- 📌 ポイント2: 説明テキスト
- 📌 ポイント3: 説明テキスト
```

### パターン3: 2カラムレイアウト
```markdown
# 比較: A vs B

<div style="display: flex; gap: 60px;">
<div>

### オプションA
- メリット1
- メリット2

</div>
<div>

### オプションB
- メリット1
- メリット2

</div>
</div>
```

### パターン4: 画像 + テキスト
```markdown
![bg right:40%](image.png)

# 見出し

説明テキストをここに記載。
画像と並べることで視覚的に理解しやすくなる。
```

### パターン5: 引用・ハイライト
```markdown
<!-- _backgroundColor: #F8FAFC -->

> 💡 **重要なメッセージ**
>
> ここに強調したい内容を記載する

```

## 6. 絵文字・アイコン活用ガイド

### 推奨絵文字カテゴリ

| 用途 | 絵文字例 |
|------|----------|
| 開始/導入 | 🚀 🎯 ✨ |
| ポイント/要点 | 📌 💡 ⭐ |
| 注意/警告 | ⚠️ 🔔 ❗ |
| 成功/完了 | ✅ 🎉 👍 |
| 質問/Q&A | ❓ 🤔 💬 |
| データ/分析 | 📊 📈 🔍 |
| 時間/プロセス | ⏰ 📅 🔄 |
| 人/チーム | 👥 🙋 🤝 |

### 絵文字使用ルール
- 見出しの先頭に1つ
- 箇条書きの区切りに使用
- 過度な使用は避ける（1スライド3-5個まで）

---

## 7. アンチパターン（避けるべきこと）

### ❌ やってはいけないこと

1. **テキストの詰め込み**
   - 1スライドに情報を詰め込みすぎない
   - 箇条書きは最大5項目

2. **単調なレイアウト**
   - 全てのスライドが同じ構造
   - 視覚的変化がない

3. **不統一なスタイル**
   - フォントサイズがバラバラ
   - 色の使い方が一貫していない

4. **装飾過多**
   - アニメーション過多（Marpでは制限あり）
   - 無意味な装飾

5. **低コントラスト**
   - 読みにくい色の組み合わせ
   - 薄い文字色
